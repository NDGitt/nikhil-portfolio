Setup Guide: AI Email Manager

Transform your chaotic inbox into an organized machine using AI automation.

The complete automation: Gmail Trigger → AI Classification → Labeling → Drafting Replies

This workflow acts as your personal executive assistant. It reads every new email, categorizes it into smart buckets (like "Job Apps" or "Newsletters"), applies the right label in Gmail, and even drafts a professional reply for emails that need your attention.

How the Workflow Works

1

Gmail Trigger runs every minute, checking for new emails

2

Pre-filter excludes emails already labeled (to avoid re-processing)

3

Main Email Agent (AI) reads the email content and metadata

4

AI categorizes and chooses ONE category based on your custom rules

5

Applies the Gmail label automatically

6

Optional: IF label is "To Respond" → Draft Agent creates a professional reply draft

Understanding the Modular Design

This workflow has 3 independent sections that work together:

Setup Nodes

Generate your custom prompt (disabled after first use)

Main Workflow

Label incoming emails (always active)

Eval Workflow

Test accuracy (optional, separate)

Why modular? You can enable/disable features independently. Want to turn off draft generation? Just disable that section. Want to test different rules? Only run the eval workflow.

2

Generate Your Custom Prompt

This is a ONE-TIME setup step. After completing this, you'll disable these nodes and they won't run again unless you want to change your categories.

Before running the main workflow, you need to generate a custom prompt based on YOUR specific label usage and rules. This is what makes the AI understand how YOU want emails categorized.

1

Find the Setup Section: In the n8n workflow canvas, look for the disabled nodes at the top (they'll be grayed out). These include: "When chat message received", "Get many labels", "Code in JavaScript", "Message a model", and "Step 2 Output".

2

Enable These Nodes: Right-click each node and select "Enable" (or click the toggle in the node settings).

3

Open the Chat Panel: Look for the chat icon on the left sidebar of n8n (it looks like a message bubble). Click it to open the chat interface.

4

Describe Your Label Rules: Type your instructions in the chat panel. Tell the AI what each label means and when it should be used.

Example prompt you could type:

"I want to use Marketing for promotional emails from brands I use. Job Applications should be for updates about positions I've already applied to. New Job Opportunities is for recruiter outreach about positions I haven't applied to yet. To Respond is only for emails from real people asking me direct questions that need my input. Meetings is for calendar invites and meeting requests. Newsletters is for content digests I've subscribed to. Cold Email is for unsolicited sales pitches. Events is for conference and workshop invitations."

5

Execute the Workflow: Click "Test Workflow" or "Execute Workflow". The AI will read your Gmail labels, combine them with your instructions, and generate two outputs.

6

Save the Outputs: Open the "Step 2 Output" node execution. You'll see two fields:

  • prompt_block - Your full custom categorization rules (this is what the AI will use to make decisions)
  • label_mapping - Gmail Label IDs matched to your category names

Copy both of these - you'll need them in Phase 3.

7

Disable the Setup Nodes: Right-click each setup node again and select "Disable". This prevents them from running with the main workflow.

What just happened? The AI analyzed your Gmail labels, understood your rules, and generated a custom system prompt that teaches the Main Email Agent exactly how YOU want emails categorized. This makes the workflow personalized to your inbox needs.

3

Configure the Main Agent

Now you'll plug your custom prompt and label mappings into the Main Email Agent. This is where your personalized rules come to life.

1

Open the Main Email Agent Node: In the workflow canvas, find and click the node called "Main Email Agent".

2

Replace the Categories Section: Scroll through the system prompt until you find the section that starts with:

### CATEGORIES & LABELS

Select everything from that header down to (but NOT including) the next header:

### GMAIL LABEL IDS - USE THESE ONLY

Replace that entire block with your prompt_block from Phase 2.

3

Replace the Label IDs Section: Continue scrolling until you find:

### GMAIL LABEL IDS - USE THESE ONLY

You'll see a list like:

- JOB_APPLICATION_UPDATE → "Label_124116732797058344"
- NEW_JOB_OPPORTUNITIES → "Label_5342224488745394684"

Replace this section with your label_mapping from Phase 2. Make sure the format matches (CATEGORY_NAME → "Label_ID").

4

Save the Node: Click "Save" or close the node editor to apply your changes.

Why this matters: The Main Agent now knows YOUR specific categories, YOUR rules for when to apply them, and the exact Gmail Label IDs to use. This is what makes the AI accurate and personalized to your inbox.

Want Different Categories?

The beauty of this workflow is that it's fully customizable. You're not stuck with the default categories!

Create whatever labels you want in Gmail (e.g., "Invoice", "Family", "Urgent", "Newsletter Digest")

Go back to Phase 2 and run the setup nodes again with your new label descriptions

Copy the new prompt_block and label_mapping outputs

Paste them into the Main Email Agent (Phase 3) to update your rules

Example custom categories: "Invoice/Billing" for accounting emails, "Family/Personal" for home life, "Client Updates" for work projects, "Subscriptions" for recurring charges, "Travel" for booking confirmations.

6

Optional: Enable Draft Agent

This feature is optional. The Draft Agent automatically writes professional reply drafts for emails that need your response.

How the Draft Agent Works:

1

After the Main Agent labels an email, it checks: "Is this labeled 'To Respond'?" (Label_2)

2

If YES → The workflow triggers the "Email Draft Agent"

3

The Draft Agent reads the email thread, understands the question or request, and writes a professional reply

4

The draft appears in your Gmail Drafts folder, ready for you to review and send

Finding the Control Node:

Look for the node called "Is a response expected?" in the workflow. This is the conditional gate that controls whether drafts get created.

To disable draft generation entirely, you can disable the "Email Draft Agent" and "Create a draft in Gmail" nodes.

Why separate agents? The Main Agent (categorization) runs fast and cheap using GPT-4o-mini. The Draft Agent (writing) only triggers when needed, making the workflow efficient and cost-effective.

Testing with Evaluations (Advanced)

The workflow includes an optional evaluation system to test how accurately the AI categorizes emails. This is useful when you want to refine your rules or test new categories.

How Evaluations Work:

The bottom section of the workflow (nodes starting with "When fetching a dataset row") is a separate testing system. It reads test cases from a Google Sheet, runs the Main Agent on them, and compares predicted labels to your expected labels.

Connecting the Eval Workflow:

When running evaluations, you need to connect the nodes correctly. This is how you should wire the eval section to test your Main Agent:

The eval section should be connected as shown to properly test email categorization accuracy.

Google Sheet Setup:

Create a Google Sheet with these columns:

email_id
Gmail message ID
email_json
Full email JSON string
expected_label_id
The label you expect
expected_has_draft
TRUE or FALSE

When to Use Evals:

  • You notice the AI is consistently mislabeling certain types of emails

  • You've updated your categories and want to test before going live

  • You want to measure accuracy improvements over time

Note: The eval workflow runs separately from the main workflow. You trigger it manually when you want to test, and it writes results back to your Google Sheet showing predicted vs expected labels.

Troubleshooting Common Issues

Workflow doesn't trigger

✓ Check that the workflow is Active (toggle in top right should be ON)

✓ Verify the Gmail Trigger is set to run every minute

✓ Check that Gmail OAuth2 credentials are connected

No labels applied to emails

✓ Verify label IDs in Phase 3 match your actual Gmail label IDs

✓ Check the Main Agent's execution output - does it show a tool call?

✓ Ensure the format is correct: Label_123456789

AI applies wrong labels

✓ Go back to Phase 2 and refine your rule descriptions

✓ Be more specific about "when to apply" and "when NOT to apply"

✓ Use the Eval workflow to test and improve accuracy

Drafts not created

✓ Check that "To Respond" label ID is correct (should be Label_2 or your custom ID)

✓ Verify the "Is a response expected?" node is enabled

✓ Check that the Draft Agent and "Create a draft in Gmail" nodes are enabled

Nodes show red triangles

✓ Red triangles mean credentials aren't connected

✓ Click each node with a triangle → Select your credential from dropdown → Save

✓ Make sure you've created the credentials first (Phase 4)
